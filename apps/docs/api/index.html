PUBLIC ASSET API · VERSION 1

Fetch one scene.
Mount the real renderer.

Atmoscene API v1 is a static, cacheable JSON API designed for GitHub Pages and any browser, build pipeline, or edge runtime that can use HTTP. Every one of the 3,552 scene combinations has a stable URL. The optional JavaScript SDK loads the same renderer and CSS used by the Atmoscene library.

import { AtmosceneClient } from 'https://afnan-hossain.github.io/atmoscene/api/v1/client.mjs';

const atmoscene = new AtmosceneClient();
const scene = await atmoscene.scene('day.sunset.none.rain');
await atmoscene.mount('#weather-scene', scene);

Open the working API consumer or inspect the machine-readable manifest.

CANONICAL IDENTIFIERS

Scene ID grammar

{mode}.{phase}.{moonPhase-or-none}.{weather}

day.sunset.none.rain
night.midnight.full.aurora-strong
night.moonrise.waxing-crescent.tropical-cyclone-night

IDs are lowercase, dot-separated and stable within API v1. Day scenes always use none for the moon segment. Night scenes include a named lunar state. A scene response also includes family, category, effect, wind profile, original renderer attributes and preview links.

HTTP ENDPOINTS

Static URLs, normal fetch

GET /api/v1/health.json
GET /api/v1/manifest.json
GET /api/v1/scenes/index.json
GET /api/v1/scenes/by-id/{sceneId}.json
GET /api/v1/collections/{collectionId}.json
GET /api/v1/lookups/weather-family.json
GET /api/v1/checksums.json
GET /api/v1/schemas/scene.schema.json
GET /api/v1/schemas/resolve-input.schema.json

Collection IDs are core, snow-ice, aurora-polar, tropical-marine, aerosols and rare-celestial. Prefer a collection endpoint for discovery and a by-ID endpoint in production. The complete index is intended for search tools, build pipelines and offline catalogues.

const response = await fetch(
  'https://afnan-hossain.github.io/atmoscene/api/v1/scenes/by-id/day.sunset.none.rain.json'
);
if (!response.ok) throw new Error(`HTTP ${response.status}`);
const scene = await response.json();

ZERO-DEPENDENCY SDK

Client methods

const client = new AtmosceneClient({ baseUrl, fetchImpl });

client.health({ signal })
client.manifest({ signal })
client.collection(collectionId, { signal })
client.scene(sceneId, { signal })
client.sceneIndex({ signal })
client.resolve({ mode, phase, moonPhase, weather }, { signal })
client.loadRuntime()
client.mount(target, sceneOrId, { label, replace })
client.unmount(target)

Responses are cached in memory by URL. Pass an AbortSignal to cancel fetches. Supply baseUrl for a self-hosted copy or local development server, and fetchImpl for compatible non-browser runtimes.

Classic script

<script src="https://afnan-hossain.github.io/atmoscene/api/v1/client.js"></script>
<script>
  Atmoscene.create().then(async (client) => {
    await client.mount('#sky', 'night.midnight.full.clear-night');
  });
</script>

RENDERING CONTRACT

Mount without duplicating Atmoscene

mount() fetches scene metadata when given an ID, loads the versioned registry, engine, renderer and stylesheet once, then creates markup with the canonical compositor. The target receives role="img", an accessible label and data-atmoscene-id.

<div id="sky" style="position:relative;min-height:420px"></div>
<script type="module">
  import { AtmosceneClient } from '/api/v1/client.mjs';
  const client = new AtmosceneClient({ baseUrl: location.origin });
  await client.mount('#sky', 'day.sunrise.none.partly-cloudy');
</script>

The host controls dimensions. Give it a position context and an explicit height or aspect ratio. Atmoscene’s reduced-motion rules remain active automatically.

ERROR MODEL

Typed, inspectable failures

The SDK throws AtmosceneError with code, message and details. Codes include INVALID_FETCH, ABORTED, NETWORK_ERROR, HTTP_ERROR, DOM_REQUIRED, STYLE_LOAD_ERROR, SCRIPT_LOAD_ERROR, RUNTIME_MISSING, TARGET_NOT_FOUND and INVALID_SCENE.

try {
  await client.mount('#sky', sceneId);
} catch (error) {
  if (error.code === 'HTTP_ERROR' && error.details.status === 404) {
    showFallbackArtwork();
  }
}

DELIVERY

Load one scene, not the catalogue

  • Use a by-ID endpoint after selecting a scene.
  • Cache the immutable scene response in your application or CDN.
  • Load the runtime only when a scene enters the viewport.
  • Reuse one client instance so manifest and responses stay cached.
  • Avoid the full index on a normal weather screen.
  • Respect prefers-reduced-motion; the library supplies static states.

FRAMEWORK ADAPTERS

React, Vue, Svelte, Web Components and SSR

Mount after the component’s DOM node exists and call unmount() during cleanup. Do not call mount() during server rendering; fetch scene JSON on the server if useful, then mount the visual after hydration.

// React effect sketch
useEffect(() => {
  const client = new AtmosceneClient();
  client.mount(ref.current, sceneId);
  return () => client.unmount(ref.current);
}, [sceneId]);
// Vue / Svelte lifecycle sketch
onMounted(() => client.mount(element, sceneId));
onBeforeUnmount(() => client.unmount(element));

// A Web Component can call the same pair from
// connectedCallback() and disconnectedCallback().

For SSR, render the text forecast and a stable scene container on the server. Import the SDK only in the browser. This avoids DOM access during rendering and prevents hydration mismatches.

ACCESSIBILITY

Meaning before motion

Pass a concise label when the visual communicates information. If adjacent text already conveys the same weather, set the host to decorative after mounting with aria-hidden="true". Never rely on animation or color alone for severe-weather warnings. Preserve the library’s reduced-motion behavior and ensure text placed over a scene meets contrast requirements.

STABILITY, HOSTING & PRIVACY

Versioned paths and release rules

Breaking changes receive a new major path such as /api/v2/. Compatible metadata additions may appear in v1. Consumers should ignore unknown response fields. Generated output is validated against the canonical registry count so API, catalogue and homepage cannot silently disagree.

GitHub Pages limits

API v1 is static. URLs return published JSON files; there is no server-side query execution, secret storage, account system or uptime SLA. Use resolve() to construct a stable by-ID request, mirror the apps/api/v1 directory for self-hosting, or place it behind your own CDN. Check health.json and pin the /v1/ path in production.

Privacy and security

The asset API and SDK do not request location, use cookies, collect analytics, execute user strings as code or require API keys. The optional homepage weather demo is a separate consumer with its own provider requests. Treat scene labels as data and keep your application’s normal Content Security Policy, dependency review and request timeouts.
